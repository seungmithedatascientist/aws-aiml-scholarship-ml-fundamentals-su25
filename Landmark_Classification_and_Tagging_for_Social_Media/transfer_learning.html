Question: Outline the steps you took to get to your final CNN architecture and your reasoning at each step. Describe why you think the architecture is suitable for the current problem.

Answer:

For the transfer learning approach, I selected ResNet18 as the backbone architecture because it has been pretrained on ImageNet, which contains millions of images across 1000 categories, giving it robust feature extraction capabilities for visual recognition tasks. Also, I strategically chose to freeze all pretrained layers except the final classification layer because the early and middle layers of ResNet18 have already learned to detect edges, textures, shapes, and high-level visual patterns that are universally useful for image recognition. By keeping these frozen, it can keep this trained knowledge while preventing overfitting on our relatively smaller landmark dataset.

I think this architecture is suitable for landmark classification because landmarks often have similar visual elements to ImageNet categories: Buildings have architectural features similar to houses and towers in ImageNet, natural landmarks share characteristics with mountains and landscapes, and monuments have structural elements comparable to various objects in the pretrained dataset. By replacing only the final fully connected layer with a new one mapping from 512 features to the current 50 landmark classes, the model creates a highly efficient learning setup where the model only needs to learn how to combine the already-extracted high-level features into landmark-specific classifications. This approach dramatically reduces the number of trainable parameters from millions to just around 25,000 (512 50), making training faster, more stable, and less vulnerable to overfitting. I expect the transfer learning approach should achieve significantly better accuracy than training from scratch, likely reaching 70-90% accuracy compared to the ~50% achieved with the custom CNN, while requiring fewer training epochs and being more robust to the limited size of the landmark dataset.

from src.predictor import Predictor
from src.helpers import compute_mean_and_std

# First let's get the class names from our data loaders
class_names = data_loaders["train"].dataset.classes

# Then let's move the model_transfer to the CPU
# (we don't need GPU for inference)
model_transfer = model_transfer.cpu()
# Let's make sure we use the right weights by loading the
# best weights we have found during training
# NOTE: remember to use map_location='cpu' so the weights
# are loaded on the CPU (and not the GPU)
model_transfer.load_state_dict(
    torch.load("checkpoints/model_transfer.pt", map_location="cpu")
)

# Let's wrap our model using the predictor class
mean, std = compute_mean_and_std()
predictor = Predictor(model_transfer, class_names, mean, std).cpu()

# Export using torch.jit.script
scripted_predictor = torch.jit.script(predictor)
scripted_predictor.save("checkpoints/transfer_exported.pt")
